The Zebra Ontology is a small ontology project that I've been working on for a few hours. In the Applied Ontology course at U. Buffalo, the Zebra Puzzle was put forth as a challenge. I published my work here as a sample of my capabilities with Protege and building ontologies. 

You'll also find here the current version of my MotoGP Ontology project. It originally started out as an attempt to map out the causes of accidents, as well as their frequency and location, with LCR and Castrol Honda as the focus. Unfortunately, that version of the project is on hold since it is difficult to access the particular data of accidents involving Honda racers (e.g., I could not find data on the exact tire temperature of the Hondas involved in accidents). So, the current incarnation of this project is a resource ontology that racing teams or companies could take and add data into in order to draw conclusions about accidents and motorbike malfunctions and errors. My hope is that a mature version of this ontology contributes to making motorbike racing safer, and help racing teams to improve their bikes. 

As it stands, the current version needs much work. In particular, classes need definitions to be written up, which will be the next phase of the project before I continue on to adding newer classes.
